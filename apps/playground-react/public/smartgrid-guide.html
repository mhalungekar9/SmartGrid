Install and Import

In the workspace playground, import the React package directly. The package re-exports core types like Column. 

import { SmartGrid, type Column } from "@smartgrid/react";

Basic Grid

Define rows and typed columns, then pass them to SmartGrid. The grid renders immediately with header, body, toolbar, and status bar behavior. 

interface Employee {
  id: number;
  name: string;
  city: string;
  department: string;
  score: number;
}

const rows: Employee[] = [
  { id: 1, name: "John", city: "London", department: "Ops", score: 92 },
  { id: 2, name: "Alice", city: "Paris", department: "Product", score: 87 },
];

const columns: Column<Employee>[] = [
  { id: "name", field: "name", headerName: "Name", width: 220 },
  { id: "city", field: "city", headerName: "City", width: 180 },
  { id: "score", field: "score", headerName: "Score", width: 120 },
];

<SmartGrid columns={columns} rows={rows} />
  • Required props: columns and rows.
  • Typing: Column<Employee> keeps fields aligned with row data.
  • Rendering: valueGetter, valueFormatter, and cellRenderer customize cell output.

Column Options

Most SmartGrid behavior is switched on from the column definition. Use stable id values because sorting, filtering, pinning, and width state all refer to column ids. 

const columns: Column<Employee>[] = [
  {
    id: "score",
    field: "score",
    headerName: "Score",
    width: 130,
    minWidth: 90,
    maxWidth: 220,
    sortable: true,
    filter: "number",
    editable: true,
    editor: "number",
    pinned: "right",
    valueFormatter: (value) =>
      typeof value === "number" ? value.toFixed(1) : String(value ?? ""),
  },
];

Filtering

SmartGrid supports text, number, date, set, and multi-condition filter models. Add filter to columns, or pass a controlled columnFilters object. 

const columns = [
  { id: "name", field: "name", headerName: "Name", filter: "text" },
  { id: "age", field: "age", headerName: "Age", filter: "number" },
  { id: "city", field: "city", headerName: "City", filter: "set" },
  { id: "hired", field: "hired", headerName: "Hired", filter: "date" },
] satisfies Column<Employee>[];

<SmartGrid
  columns={columns}
  rows={rows}
  columnFilters={{
    city: { type: "set", operator: "in", values: ["London"] },
    score: { type: "number", operator: "gte", value: 80 },
  }}
/>
  • Toolbar Filters: opens the full filter panel.
  • Header menu: can open or clear a column filter.
  • Custom predicate: pass filter: { predicate: customPredicate } for custom matching.

  quickFilterText="engineering"
/>
  • Use the Quick filter toolbar input for uncontrolled searching.
  • Pass quickFilterText when a parent component owns the query.
  • Use the Find input and Next button to jump between matching cells.

Pivoting

Pivoting turns unique values from pivotBy into generated columns and aggregates value columns inside each groupBy bucket. If no value columns are provided, it falls back to counts. 

<SmartGrid
  columns={columns}
  rows={rows}
  groupBy="department"
  pivotBy="city"
  pivotValueColumns={["score"]}
  pivotAggregation="avg"
  pageSize={10}
/>
  • Pivot columns: unique city values become columns.
  • Rows: each department becomes one pivot row.
  • Aggregations: sum, avg, count, min, and max.
  • Sorting: generated pivot columns can be sorted and resized.

Editing

Mark a column as editable and choose an editor. Users can double-click, press Enter, or press F2 to edit cells. 

const columns: Column<Employee>[] = [
  { id: "name", field: "name", headerName: "Name", editable: true, editor: "text" },
  { id: "age", field: "age", headerName: "Age", editable: true, editor: "number" },
  { id: "hired", field: "hired", headerName: "Hired", editable: true, editor: "date" },
  { id: "active", field: "active", headerName: "Active", editable: true, editor: "checkbox" },
  {
    id: "department",
    field: "department",
    headerName: "Department",
    editable: true,
    editor: {
      type: "advancedSelect",
      values: ["Operations", "Product", "Engineering"],
      allowCustomValue: false,
    },
  },
];
  • Supported editors: text, number, date, checkbox, largeText, select, and advancedSelect.
  • Commit: Enter or blur commits the edit.
  • Cancel: Escape leaves the value unchanged.
  • Callback: use onCellValueChange to react to edits.

Selection

Row selection and active cell state are built in. Range selection is enabled by default and can be disabled with enableRangeSelection. 

<SmartGrid
  columns={columns}
  rows={rows}
  enableRangeSelection
/>
  • Click a cell to move the active cell.
  • Use Shift with arrow keys or cell clicks to extend a range.
  • The status bar shows active cell, selected row, and range size.

Clipboard and Fill

Copy and paste work with the active cell or selected range. The Fill toolbar action repeats the starting cell value across a selected range. 

<SmartGrid
  columns={columns}
  rows={rows}
  enableFillHandle
/>
  • Ctrl or Cmd + C copies the current cell or range.
  • Ctrl or Cmd + V pastes tab-separated values.
  • Pasting into numeric cells parses numbers when possible.

Transactions

Use transaction to add, update, or remove rows from the grid state. Provide getRowId for stable matching. 

<SmartGrid
  columns={columns}
  rows={rows}
  getRowId={(row) => row.id}
  transaction={{
    add: [{ id: 9, name: "Rhea", city: "Berlin", department: "Finance", score: 89 }],
    update: [{ id: 2, name: "Alice", city: "Paris", department: "Product", score: 91 }],
    remove: [{ id: 3, name: "Bob", city: "New York", department: "Support", score: 79 }],
  }}
/>

Server Side State

onServerSideOperation reports sort, filter, selection, page, group, pivot, tree, detail, and transaction state. Use it to mirror grid operations to a backend. 

<SmartGrid
  columns={columns}
  rows={rows}
  getRowId={(row) => row.id}
  onServerSideOperation={(state) => {
    console.log(state.sortModel);
    console.log(state.filterModel);
    console.log(state.pivotBy, state.pivotValueColumns);
  }}
/>
Example output Grid operations are reported as structured state 
{
  sortModel: [{ columnId: "score", direction: "desc" }],
  filterModel: { city: { type: "set", values: ["London"] } },
  pageIndex: 0,
  groupBy: "department",
  pivotBy: "city"
}

Locale Text

Pass localeText to replace supported toolbar labels. Missing keys fall back to built-in English labels. 

<SmartGrid
  columns={columns}
  rows={rows}
  localeText={{

    find: "Find cell",

/>
